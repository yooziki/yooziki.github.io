渲染管线

主要功能：决定在给定虚拟相机、三位物体、光源、照明模式，以及纹理等诸多条件的情况下生成或绘制一副二维图像的过程

应用阶段

​ 将需要在屏幕上显示出来绘制的几何体，也就是绘制图元，比如点、线、矩形等输入到绘制管线的下一个阶段。

​ 具体包括图元的顶点数据、摄像机位置、光照纹理等参数

几何阶段

需要将顶点数据最终进行屏幕映射

这其中需要：

  • 将各个图元放入到世界坐标系中，也就是进行模型变换
  • 根据光照纹理等计算顶点处材质的光照着色效果
  • 根据摄像机的位置、取景范围进行观察变换和裁剪
  • 最后进行屏幕映射，也就是把三维模型转换到屏幕坐标系中

光栅化阶段

​ 光栅化部分的输入是经过变换和投影后的顶点、颜色以及纹理坐标，它的工作是给每个像素正确配色，以便绘制整幅图形。

​ 由于输入的是三角形顶点，所以需要根据三角形表面的差异，逐个遍历三角形计算各个像素的颜色值。之后根据其可见性等进行合并得到最后的输出。

GLSL与OpenGL的通信

GLSL数据类型

  • 标量：
    整数(int)、无符号数(uint)、布尔类型(bool)、浮点数(float)

  • 矢量：一个矢量可以有2、3、4个分量
    vec4 a=vec4(1.0,2.0,3.0,4.0)

  • 矩阵：
    mat2 m=mat2(1.0,2.0,3.0,4.0); m=[1234]m=\begin{bmatrix} 1 & 2 \\ 3 & 4 \\ \end{bmatrix}m=[13​24​]

  • 结构和数组：结构体的成员或者数组的基类型可以为任意的数据类型

  • 循环结构：for、while、do-while

  • 选择结构：if-then、if-then-else, GLSL3.0版本引入了switch结构

EBO、VBO和VAO

  • EBO
    Element Buffer Object，也叫IBO:Index Buffer Object
    索引缓冲区对象，这个缓冲区主要用于存储顶点的索引信息

  • VBO
    Vertex Buffer Object顶点缓冲区对象，主要用于存储顶点的各种信息。
    好处：模型的顶点信息放进VBO，这样每次画模型的时候，数据不用再从CPU的势力范围内读取，而是直接从GPU的显存读取，提高效率

  • VAO

    Vertex Array Object顶点数组对象
    VAO是一个保存了所有顶点数据属性的状态结合，它存储了顶点数据的格式以及顶点数据所需的VBO对象的引用

​ 首先，使用glfwInit初始化GLFW，然后使用glfwWindowHint(<参数选项(枚举)>，<参数内容>)来配置GLFW。它的第一个选项是选择参数的名称，可以从GLFW_为开头的枚举选项中选择；第二个选项接受一个参数具体值，是整形类型。此阶段出现undefined reference则代表GLFW库并未被顺利连接。

​ 由于是基于OpenGL3.3进行讨论的的，将版本主号(Major)和次版本号(Minor)都设置为3(高版本对低版本兼容，最低3.3,使用的GLAD版本不能高于GPU的OpenGL版本)，同样我们明确告诉GLFW使用的是核心模式(Core-profile)。明确告诉GLFW我们使用核心模式意味着我们只能使用OpenGL功能的一个子集(没有我们已不再需要的向后兼容特性)。如果是Mac OS X系统，还需要添加注释段代码这些配置才会起作用。

我们还可以查看GLFW当前使用的版本号

​ 接下来创建窗口，窗口存放了所有和窗口相关的数据，而且会被GLFW的其他函数频繁地使用到。

glfwCreateWindow(<宽>,<高>,<窗口名字符串>,<是否使用全屏模式>，<时候共享上下文窗口>)这个函数会返回一个GLFWwindow对象，我们会在其他的GLFW操作中使用到。创建完窗口，我们就可以通知GLFW将我们窗口的上下文设置为当前线程的主上下文了。

OpenGL可以看作一个大的状态机，OpenGL的状态即OpenGL的上下文。

GLAD

​ GLAD是用来管理OpenGL的函数指针的，所以在调用如何OpenGL的函数之前我们需要初始化GLAD。

​ 我们给GLAD传入了用来加载系统相关的OpenGL函数指针地址的函数，GLFW给我们的是glfwGetProcAddress，它根据我们编译的系统(CMAKE)定义了正确的函数。

渲染

​ 程序应该在我们主动关闭之前不断绘制图形，并且能够接受用户输入。因此，我们需要在程序中添加一个while循环，我们可以把它称之为渲染循环(Render Loop)，它能在我们狼GLFW退出前一直保持运行。

  • glfwWindowShouldClose()在循环开始之前检查GLFW时候要求被退出。
  • glfwPollEvents()函数检查时候触发什么事件，更新窗口的状态，并且调用对应的回调函数（可以通过回调方法手动设置）
  • glfwSwapBuffer函数会交换颜色缓冲（它是一个存储着GLFW窗口每一个像素颜色值的大缓冲，它在这一迭代中被用来绘制，并且会作为输出显示在屏幕上
  • 双缓冲
    应用程序使用单缓冲绘图时可能会存在图像闪烁的问题。这是因为生成的图像不是一下子被绘制出来的，而是按照从左到右，由上而下逐像素地绘制而成的。最终图像不是在瞬间显示给用户，而是通过一步一步生成的，这会导致渲染的结果很不真实。为了规避这个问题，我们应用双缓冲渲染窗口应用程序。钱荒城保存着最终输出图像，他会在屏幕上显示；而所有的渲染指令都会在后缓冲上绘制。当所有的渲染指令执行完毕后，我们交换前缓冲和后缓冲，这样图像就立即呈现出来，之前提到的不真实感就消除了。

最后

当渲染循环结束后，我们需要正确释放/删除之前的分配的所有资源。我们可以在main函数的最后调用glfwTerminate函数来完成

这样就可以清理所有的资源并正确退出应用程序。现在你可以尝试编译并正确运行你的应用程序了。(黑色窗口)

附加

我们要把所有的渲染操作放在渲染循环中，因为我们想让这些渲染指令在每次渲染循环迭代的时候都能被执行。

​ 为了测试这一切都正常工作，我们使用一个自定义的颜色清空屏幕。在每个新的渲染第二代开始的时候我们总是希望屏幕清空，否则我们仍能看见上一次迭代的渲染结果。我们可以通过调用glClear函数来清空屏幕的颜色缓冲，它接受一个缓冲位(Buffer Bit)来指定要清空的缓冲，可能的缓冲位有GL_COLOR_BUFFER_BIT,GL_DEPTH_BUFFER_BIT和GL_STENCIL_BUFFER_BIT。由于现在我们还关心颜色值，所以我们只清空颜色缓冲。

DDA直线生成算法特点

​ 在一个迭代算法中，如果每一步的x、y值是用前一步的值加上一个增量来获得的，那么，这种算法就称为增量算法。因此，DDA算法是一个质量算法。

  • 优点：DDA算法直观、易实现
  • 缺点：有浮点数和浮点运算，效率不高

标准椭圆是关于x、y轴对称的，计算的时候只计算第一象限

原理

椭圆的方程

x2a2+y2b2=1长半轴a，短半轴b\frac{x^{2}}{a^{2}}+\frac{y^{2}}{b^{2}}=1 \\ 长半轴a，短半轴b a2x2​+b2y2​=1长半轴a，短半轴b

坐标平面分为三个部分

第一象限的四分之一椭圆分为上半部分和下半部分，划分点是圆弧斜率为-1处，微分公式得到

上半部分y分量>x分量，下部分y分量<x分量，分界处y分量=x分量

多边形表示

对多边形的描述方式：

顶点表示

输入：是多边形的顶点序列P1P_1P1​到PnP_nPn​

输出：最佳逼近这个多边形的像素点集

X-扫描线算法

  1. 确定多边形所占有的最大扫描线数，得到多边形顶点的最小和最大y值(yminy_{min}ymin​和ymaxy_{max}ymax​)
  2. 从y=yminy=y_{min}y=ymin​到y=ymaxy=y_{max}y=ymax​，每次用一条扫描线进行填充。
  3. 对每一条扫描线填充的过程可分为四个步骤
    1. 求交
    2. 排序
    3. 交点配对
    4. 区间填色

几个讨论的问题：

  1. 当扫描线与多边形顶点相交时，交点的取舍问题
    策略：交点个数= 构成这个顶点的两条边位于扫描线上方的条数
  2. 绘图时的下闭上开，左闭右开原则
    中心会偏移半个像素
  3. 算法效率问题
    对每一条扫描线填充过程——求交和排序
    • 求交
      对于一条扫描线，只需要与有效边求交
      扫描线和直线在Y方向上都具有连贯性，将问题转化为增量计算：yi+1=yi+1，xi+1=xi+1/ky_{i+1}=y_i+1，x_{i+1}=x_i+1/kyi+1​=yi​+1，xi+1​=xi​+1/k
    • 排序
      不需要每次都进行排序，有效边失效时删去有效边即可，新有效边时需要重新进行排序

Y向连贯性算法

由上一个算法改进得来，改进核心:Y方向上具有连贯性

####　数据结构

边表桶：每条多边形的边放入首次出现的桶中(yminy_{min}ymin​)

  • x:表示该边起点与x扫描线的交点
  • ymaxy_{max}ymax​:表示该边最高的y值
  • 1/k1/k1/k:表示当y增加1时x的增量
  • next:指向下一条边

在同y值下，x相同时，1/k1/k1/k的值越小越靠前，然后按照x值递增的顺序进行排序

算法描述：

  1. 建立一个空的有效边表
  2. 将边表桶中的yminy_{min}ymin​读入有效边表，两两组合进行填充
  3. y=yi+1y=y_i+1y=yi​+1，当结点中ymax=yy_{max}=yymax​=y满足时删去结点，将此结点的next交给它的前一个结点，两两组合，并且x值通过增量算法获得
  4. 当y=yiy=y_iy=yi​处的边表桶中有新边时，需要重新进行排序
  5. 重复步骤3和4直至y=ymaxy=y_{max}y=ymax​，多边形填充结束

优点

  • 采用增量计算的方法进行交点计算
  • 仅在新边加入时排序(边数<<扫描线数)

缺点

  • 桶表、链表的维护开销

边标志算法

  1. 用一种特殊的颜色在帧缓器中将多边形的边界勾画出来
  2. 将作色的像素点依x坐标递增的顺序两两配对
  3. 将每一对像素所构成的扫描区间内的所有像素置为填充色
    1. 打标记
      对多边形每条边进行直线扫描转换
      (将多边形边界所经过的像素打上印记)
    2. 填充
      扫描线以x递增的方式检测边标记(初始Inside值为假)
      遇见标记将Inside取反
      当Inside为真时填充，Inside为假时不填充
      顶点处理：上开下闭，左闭右开

特点

  • 避免了对帧缓存中大量元素的多次赋值，但仍然需要逐条扫描线地对帧缓存中的元素进行搜索和比较

  • 利用软件实现本算法时，速度与Y向连贯性算法相当，但本算法用硬件实现后速度会有很大提高。

区域填充

区域的定义：指已经表示成点阵形式的填充图形，它是像素集合

两种表示形式：

  • 边界表示法：把位于给定区域的边界上的像素一一列举出来的方法
  • 内点表示法：枚举出给定区域内所有像素的表示方法

区域分类：4连通和8连通

区域填充是指从区域内一个点P开始(种子点)，由内向外将填充色扩展到整个区域内的过程

种子点定义：边界表示区域内任意一点或者内点表示区域的任意一点(常见：油漆桶工具点击位置)

种子填充算法

输入：种子点坐标(x,y)，填充色和边界颜色

数据结构：栈结构

输出：最佳逼近的像素点集

  • 4-连通边界算法填充

    种子像素入栈，栈非空时重复执行三步操作：

    1. 栈顶像素出栈
    2. 将出栈像素置成填充色
    3. 检查出栈像素的4-邻接点，入其中某个像素点不是边界色且未置成填充色，则把该像素入栈

  • 8-连通边界算法填充

    种子像素入栈，栈非空时重复执行三步操作：

    1. 栈顶像素出栈
    2. 将出栈像素置成填充色
    3. 检查出栈像素的8-邻接点，入其中某个像素点不是边界色且未置成填充色，则把该像素入栈

  • 8连通边界算法不能填充4连通边界表示区域：会出现越界情况

  • 8连通泛填充可以填充4连通的内点表示区域

  • 4连通水平像素填充-减少重复入栈提高效率
    种子像素入栈，当栈非空时如下操作：

    1. 栈顶像素出栈
    2. 填充出栈像素所在扫描线行的连续像素段，直到遇到边界像素
    3. 在区间中检查与当前扫描线相邻上下两条扫描线的有关像素是否为边界像素，或已填充像素，若存在非边界、未填充边界的像素，则把每一区间的最右像素取作种子像素入栈。

反走样

走样现象

  • 斜线锯齿阶梯状边缘
  • 微小物体被忽视或者忽隐忽现闪烁

走样的本质

用离散量表示连续量引起的失真现象

反走样的本质

用于减少或消除这种效果的技术

反走样方法

  • 提高显示分辨率(4x分辨率与采样时间获得2x提升效果)，无法彻底消除走样现象

  • 过取样(后滤波)：

      graph LR
	高分辨率下取样计算 -- 对几个像素属性取平均 --> 低分辨率下的像素属性

    重叠过取样：能解决像素颜色不均匀的问题

  • 区域取样(前滤波)：根据图形在每个像素点的覆盖率来确定像素点的属性
    可以利用一种求相交区域的近似面积的离散计算方法：

    1. 将屏幕像素分割成n个更小的子像素
    2. 计算中心落在直线段内的子像素的个数m
    3. m/n为线段与像素相交区域面积的近似值

    利用滤波器对经过像素不同重叠区域进行积分，获得不同的加权值，滤波器相互重叠，有利于缩小相邻像素的灰度差

OpenGL绘制框架

void glGenBuffers(GLsizei n,GLuint * buffers):

该函数用来生成缓冲区对象的名称

第一个参数是要生成的缓冲对象的数量

第二个参数是要输入用来存储缓冲对象名称的数组

函数会在buffers里返回n个缓冲对象的名称

这里仅仅只是生成一个缓冲对象的名称，不具备如何意义，它仅仅是一个缓冲对象，类似指针变量，可以作为VAO、VAO或者其它类型的缓冲，利用glBindBuffer()赋予意义

void glCreateBuffer():

void glBindBuffer(GLenum target,GLuint buffer):

第一个参数是枚举方式，缓冲对象的类型

第二个参数是要绑定的缓冲对象名称，通过glGenBuffers()生成

如果把target绑定到一个已经创建好的缓冲对象，那么这个缓冲对象将为当前target的激活对象；但是如果绑定的buffer值为0，那么OpenGL将不再对当前target使用如何缓冲对象。

OpenGL允许我们同时绑定多个缓冲类型，**只要这些缓冲类型是不同的，换句话说，同一时间，不能绑定两个相同类型的缓冲对象。**也可以理解成对于一个类型来说，同一时间只能“激活”一个类型，否则就会产生“矛盾”

绑定缓冲类型后，所有该缓冲类型的函数调用都用来配置该目标缓冲类型，比如顶点缓冲类型GL_ARRAY_BUFFER，glBufferData通过指定目标缓冲类型来进行数据传输，而每一个目标缓冲类型使用前要提前绑定一个缓冲对象，从而赋予这个缓冲对象一个类型的意义

glBufferData(GLenum target, GLsizeiptr size, const GLvoid * data, GLenum usage)

target:指定缓冲类型，GL_ARRAY_BUFFER或者GL_ELEMENT_ARRAY_BUFFER

size：指定buffer object的data store大小

data：指定即将被拷贝进data store并初始化的数据指针，或者null，如果没有data拷贝

usage：指定你希望data store使用的模式，GL_STREAM_DRAW, GL_STATIC_DRAW, GL_DYNAMIC_DRAW

  1. glBufferData为绑定在target上的buffer object currently创建了一个新的data store。任何之前存在的data store被删除。新创建的data store将被指定size和usage.如果data参数为NULL,data store将被这个pointer指向的data初始化。
  2. usage参数提示了在GL实现中一个a buffer object’s data store将如何被访问。这使GL做出更明智的可能显著提升性能的选择。然而，如果没有，这将限制data store的准确的usage。usage可以被拆成两部分：一，被访问的频率（修改或使用），二，访问的性质。访问的频率是如下几个之一:
    STREAM
    该data store内容将被修改一次，且被使用的次数也很少；

STATIC
该data store内容将被修改一次，但会被多次使用；

DYNAMIC
该data store内容将被不断修改，且被多次使用；

访问的性质如下：
DRAW
该data store内容将被应用程序修改，且被当做GL渲染和图像设置命令的源数据。
